# 马自达车辆数据获取问题修复说明

## 问题分析和修复

### 1. 发动机转速 (Engine RPM) 问题
**问题**：发动机转速显示为空白或不正确
**原因**：DBC文件中RPM信号有缩放因子0.25，但carstate.py中没有应用
**修复**：
```python
# 修改前
ret.engineRpm = cp.vl["ENGINE_DATA"]["RPM"]

# 修改后
ret.engineRpm = cp.vl["ENGINE_DATA"]["RPM"] * 0.25
```

### 2. 刹车数据放大100倍问题
**问题**：刹车压力数值被放大了100倍
**原因**：broadcaster.py中错误地将原始值乘以100
**修复**：
```python
# 修改前
"Brake Pressure": f"{getattr(CS, 'brake', 0) * 100:.1f}%"

# 修改后
"Brake Pressure": f"{getattr(CS, 'brake', 0):.1f}"  # 原始值(0-255)
```

### 3. 油门踩下状态检测优化
**问题**：油门状态检测过于敏感，任何微小输入都被认为是踩下
**原因**：阈值设置为0，过于敏感
**修复**：
```python
# 修改前
ret.gasPressed = ret.gas > 0

# 修改后
ret.gasPressed = ret.gas >= 5  # 大于等于5%才认为踩下
```

### 4. 跟车距离 (Following Distance)
**状态**：当前逻辑正确，无需修改
**说明**：carstate.py中的转换逻辑是正确的：
```python
can_distance_setting = cp.vl["CRZ_CTRL"]["DISTANCE_SETTING"]
ret.pcmCruiseGap = 5 - can_distance_setting if 1 <= can_distance_setting <= 4 else can_distance_setting
```

## DBC信号定义参考

根据mazda_2017.dbc文件：

### ENGINE_DATA (ID: 514)
- `RPM`: 缩放因子 0.25，范围 0-8500 rpm
- `SPEED`: 缩放因子 0.01，单位 kph
- `PEDAL_GAS`: 范围 0-255，单位 %

### BRAKE (ID: 120)
- `BRAKE_PRESSURE`: 范围 0-255，原始值

### CRZ_CTRL (ID: 540)
- `DISTANCE_SETTING`: 范围 0-7，跟车距离设置

### 5. 网页显示格式优化
**问题**：刹车压力在网页上的显示格式需要相应调整
**原因**：broadcaster.py修复后发送原始值(0-255)，网页需要相应格式化
**修复**：在web.py的formatValue函数中添加刹车压力特殊处理
```javascript
// 新增刹车压力特殊格式化
if (fieldKey === 'Brake Pressure') {
    const numValue = parseFloat(value);
    if (!isNaN(numValue)) {
        const percentage = ((numValue / 255) * 100).toFixed(1);
        return `${numValue} <span style="color: #666;">(${percentage}%)</span>`;
    }
}
```
